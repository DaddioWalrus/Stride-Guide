Street tiles: stay on OSM, or move to CARTO?

Cold-start tile loading feels slow. Preconnect hints are already shipped, but the deeper constraint is the provider itself. This sheet compares the current street basemap with CARTO's, so the trade-off is decided with open eyes. The Esri satellite layer is unaffected either way.

CURRENT  tile.openstreetmap.org (OSM Standard) CANDIDATE  CARTO Voyager (+ Dark Matter) SCOPE  street layer only
Visual character

Three looks, same underlying data

All three render OpenStreetMap data — the difference is cartographic styling and delivery. Previews below are stylised recreations (live tiles can't be embedded here); check the real thing via the links in the footer before committing.

OSM Standardcurrent
stylised preview
Dense, saturated, encyclopaedic. Every POI, landuse tint and label the data has. Distinct "wiki map" look; the indigo route competes with strong greens/oranges.
CARTO Voyagercandidate · light

Muted, modern, app-like. Softer palette, fewer labels, calmer landuse tints — the route line and UI own the visual hierarchy. Sits naturally with the Horizon identity.

Head to head

The differences that matter

DimensionOSM Standard (current)CARTO Voyager / Dark Matter
Delivery speed Volunteer-funded shared servers. No commercial CDN; throttles under load; the cold-start slowness you're seeing largely lives here. Commercial CDN (Fastly). Built for app traffic; consistently faster first-tile and pan/zoom loads.
Retina sharpness No @2x tiles. On phone screens tiles render soft/blurry — every beta tester is on a phone. Native retina ({r} URL). Crisp text and lines on high-DPI displays.
Usage policy Explicitly discourages app production use. The OSMF tile policy calls heavy third-party app usage unacceptable; apps get blocked at scale. Fine for a small beta, a real liability after it. Free tier with attribution; terms need a read. CARTO basemaps are widely used free in small/non-commercial apps. If Stride Guide ever monetises, expect to need a plan (or another provider).
Dark mode None. Dark theme UI floats over a bright map at night. Dark Matter. Map can follow the app theme automatically.
Visual fit Characterful but busy. Strong colours compete with the route line and Horizon UI. Designed as an app basemap. Muted ground that lets overlays lead.
Data freshness Near-live. OSM edits appear within minutes/hours. Periodic rebuilds. Same OSM data, refreshed roughly monthly — irrelevant for street layouts, occasionally visible for brand-new paths.
Attribution © OpenStreetMap contributors © OpenStreetMap contributors © CARTO (both required)
Cost today Free (donation-run) Free at beta scale
Switch effort — ~2 lines in js/map.js (+ preconnect + attribution). CSP already allows it. Reversible in one commit.
Summarised

Pros & cons

Keep OSM Standardcurrent

  • Zero change — no new terms, no visual shift to re-verify.
  • Freshest data — edits appear near-live.
  • No second-party dependency beyond OSM itself.
  • Slow, throttled delivery — the complaint that started this.
  • Blurry on phones — no retina tiles.
  • Policy risk — OSMF discourages exactly this usage; scaling the app on these tiles invites blocking.
  • No dark map for the dark theme.

Switch to CARTOvoyager + dark matter · recommended

  • Materially faster tiles — CDN delivery, the actual fix for cold-start slowness.
  • Retina-crisp on every tester's phone.
  • Dark Matter — map follows the app's dark theme; the night-walk experience gets dramatically better.
  • Calmer cartography that flatters the Horizon UI and route line.
  • Two-line change, trivially reversible.
  • Terms to review before any commercial launch — free today, but it's their CDN, their rules.
  • Slightly staler data (monthly-ish rebuilds).
  • Look changes — worth one on-phone check before beta invites.
Recommendation: switch — Voyager in light mode, Dark Matter when the dark theme is active. It directly fixes the reported slowness, sharpens the map on every phone, and unlocks a proper dark-mode map — while the sole meaningful cost (licence review) only bites if the app later commercialises, at which point tile provision needs a proper decision anyway. Keeping OSM as a one-commit rollback makes this close to risk-free for beta.
To proceed: say "switch to Carto" for Voyager + theme-linked Dark Matter, or "Voyager only" to keep the map light in both themes. Either lands as a single reversible commit.
